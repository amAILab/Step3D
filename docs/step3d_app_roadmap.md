# Step3D App / Mini App roadmap

Цель: превратить скетчи мобильного интерфейса в рабочий продуктовый поток Step3D: выбор услуги → вводные → номер проекта → файлы → статус → расчёт.

## Уже сделано
- PWA-экран `app/` в логике скетчей: маскот, выбор 3D-печати / 3D-скана / CAD, кнопка Go.
- Выбранная услуга подставляется в заявку и сохраняется в payload.
- Есть прогресс готовности вводных, номер проекта S3D, кнопки для Telegram-файлов и кабинета.
- Viewer передаёт расчёт в App через `localStorage`.
- Request Flow покрыт smoke-тестом `scripts/check_request_flow.py`.

## Следующие обязательные шаги
1. Backend endpoint для заявок вместо зависимости только от FormSubmit.
2. Приватная загрузка файлов: фото, STL, STEP, ZIP, ссылки на Drive.
3. Реальный статус проекта: `created → needs_files → estimating → quoted → in_work → done`.
4. Telegram Mini App entrypoint: открыть тот же app внутри Telegram с передачей user/contact.
5. Авторизация/доступ к проекту по безопасной ссылке или OTP.
6. Уведомления: заявка принята, не хватает файлов, готова оценка.
7. Админ-панель/CRM для Step3D: список лидов, маршруты, заметки инженера.
8. Визуальный polish: финальный маскот/логотип, fullscreen first-run, анимации без перегруза.

## Критерий готовности MVP
- Клиент может с телефона выбрать услугу, отправить контакт и задачу, получить номер S3D, дослать файлы и открыть статус проекта.
- Step3D видит заявку в устойчивом backend/таблице и не теряет лид при сбое FormSubmit.
