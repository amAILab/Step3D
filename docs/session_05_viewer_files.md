# Session 05 — файлы и Viewer

Цель: сделать Viewer частью основного пути `viewer → app → project`, а не отдельной игрушкой/калькулятором.

## Что изменено

### Viewer

- Основной CTA теперь: `Сохранить и создать заявку`.
- Все переходы из Viewer в App сохраняют расчёт в `localStorage`:
  - верхняя кнопка `Создать заявку`;
  - hero CTA;
  - основной CTA в блоке расчёта;
  - мобильная кнопка `Заявка`.
- После сохранения показывается понятное уведомление: `Расчёт сохранён. В App он подставится в поле файлов/ссылок.`
- Header Viewer сокращён: App / Модель / Расчёт.
- Убран конкурирующий CTA `В кабинет` до создания проекта.
- Нижняя мобильная панель ведёт не только к расчёту, а сразу в заявку.
- Viewer flow теперь: модель → проверка → заявка, без ощущения отдельного продукта.

### App

- Блок `Файлы / ссылки` объясняет, что расчёт из Viewer подставится автоматически.
- Placeholder стал понятнее: `Расчёт из Viewer, ссылка на файлы или: файлы отправлю после заявки`.
- Сокращены подсказки file-intake, чтобы не перегружать форму.

### Проверки

- `python3 scripts/project_healthcheck.py` → OK
- `python3 scripts/validate_site.py` → OK
- `python3 scripts/check_request_flow.py` → OK, checks=31
- `python3 scripts/check_analytics_events.py` → OK
- `python3 scripts/check_backend_templates.py` → OK

## Итог

Viewer теперь не конкурирует с App, а подготавливает вводные для заявки. Следующая сессия — `Account`: сделать статус проекта максимально простым и честным, без демо-шумов и ожидания полноценного backend там, где его ещё нет.
