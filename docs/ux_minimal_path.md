# Step3D minimal UX path

Цель: путь как у Ollama — спокойно, понятно, без витрины всего сразу.

## Основной путь пользователя

1. Главная объясняет задачу одной фразой: оценить 3D-задачу по фото/STL.
2. Главный CTA ведёт в `app/`.
3. В `app/` пользователь выбирает один из трёх сценариев: печать, скан, моделинг.
4. Заполняет короткую заявку: контакт + задача + файлы/ссылка.
5. Получает номер `S3D-...`.
6. Досылает файлы с номером или открывает статус проекта.
7. В первом ответе Step3D даёт маршрут, срок, цену-вилку, риски и что дослать.

## Что убрано/приглушено

- Лишние CTA на первом экране.
- Технический roadmap из пользовательского пути.
- Дублирующие плитки и доказательства на app hero.
- Слишком много кнопок в кабинете.
- Viewer как конкурент главной заявке на первом экране.

## Принцип следующих правок
