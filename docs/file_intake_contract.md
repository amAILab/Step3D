# Step3D file intake contract

Цель: связать файлы клиента с номером проекта `S3D-...`, не хранить приватные модели публично и подготовить переход к backend-upload.

## MVP сейчас

На GitHub Pages нельзя безопасно хранить приватные STL/STEP/фото. Поэтому текущий поток:

1. Клиент создаёт заявку в `app/` или на главной.
2. Получает номер проекта `S3D-...`.
3. Отправляет файлы в Telegram с номером проекта или вставляет ссылку на Drive/Dropbox/Yandex Disk.
4. Step3D привязывает файлы к проекту вручную/через CRM.

## Что просить у клиента

- Фото: 2–3 ракурса, масштабный объект/линейка, проблемные зоны.
- STL/STEP/OBJ/3MF: исходник, единицы измерения, версия.
- Размеры: габариты, посадки, отверстия, допуски.
- Контекст: нагрузка, материал, внешний вид, срок, количество.

## Backend-upload MVP

Endpoint должен:

- принимать `projectId`;
- принимать файл или ссылку;
- проверять расширение и размер;
- складывать файл в приватное хранилище;
- писать метаданные в CRM/Sheet;
- возвращать JSON:

```json
{ "ok": true, "projectId": "S3D-...", "files": [{ "name": "part.stl", "status": "uploaded" }] }
```

## Разрешённые форматы

- Изображения: jpg, jpeg, png, heic/webp при поддержке backend.
- 3D: stl, step, stp, obj, 3mf.
- Архивы: zip, 7z — только после проверки размера.
- Документы: pdf, dxf, dwg при необходимости.

## Безопасность

- Не публиковать файлы в GitHub Pages.
- Не делать публичные ссылки без подтверждения клиента.
- Для B2B/закрытых проектов использовать NDA и приватное хранилище.
- Проверять MIME/расширение/размер на backend.
