# Step3D UX/UI 100/10 pass

Задача: поднять восприятие интерфейса с «рабочий MVP / 5 из 10» до цельного premium product UI.

## Что изменено

Добавлен отдельный слой `assets/ux-ui-100.css`, подключённый к главным страницам:

- `index.html`
- `app/index.html`
- `viewer/index.html`
- `account/index.html`
- `thanks/index.html`

## Принцип

Не добавлять ещё один шумный лендинг, а наоборот:

- убрать ощущение CSS-каши;
- сделать App главным продуктовым экраном;
- снизить декоративные SaaS-градиенты;
- оставить один сильный primary CTA;
- сделать UI ближе к Ollama-like: белые поверхности, чёрная primary-кнопка, спокойный фон, меньше визуального соревнования.

## Главные UX/UI решения

### 1. Premium visual layer

- Фон: тёплый off-white `#f6f5f1`.
- Поверхности: белые карточки.
- Primary CTA: чёрная кнопка.
- Secondary CTA: белая кнопка с тонкой границей.
- Синий цвет оставлен как статус/ссылка, а не как основной шум.

### 2. Главная

- Первый экран стал спокойнее и продуктнее.
- Proof/answer-card шум скрыт.
- Flow strip оставлен компактным.
- На мобильном показывается только самое важное.

### 3. Заявка Step3D / App

- App получил более сильную product-композицию.
- Avatar стал аккуратным identity-элементом.
- Услуги стали вертикальным быстрым выбором.
- Карточка проекта sticky на desktop и обычная на mobile.
- Bottom nav упрощён до 3 пунктов: заявка / файлы / статус. Viewer убран из нижней навигации как необязательный путь.

### 4. Viewer

- Viewer визуально успокоен.
- Панели и toolbar стали менее dashboard-heavy.
- Primary flow подчёркнут как следующий шаг, а не как отдельный продукт.

### 5. Account / Thanks

- Карточки и preview-состояния приведены к тому же спокойному стилю.
- Убрано ощущение тяжёлого кабинета.
