# Step3D Telegram Mini App — UX spec from internal references

Цель: сделать Mini App не «сайтом внутри Telegram», а основным рабочим пультом клиента Step3D.

Основа: голосовые указания Никиты, текущий `Step3D/app/`, `account/`, Google Sheet CRM/CMS и присланные наброски интерфейса. Сами изображения не публикуем; используем их как внутренние UX-референсы.

## Принцип инструментов

Для каждой задачи выбирать оптимальный актуальный инструмент:

- Быстрая коммуникация → Telegram Mini App + бот.
- Установка на телефон → PWA.
- Лиды и редакционный слой → Google Sheet CRM/CMS.
- Реальные статусы, файлы, приватные данные → backend/auth + хранилище файлов.
- Публикация сайта/app → GitHub diff/review → checks → deploy.

## Главный сценарий

Первое открытие Mini App даёт максимум 3 действия:

1. **Оставить заявку** — начать проект.
2. **Дослать файлы** — фото/STL/STEP/чертёж/голос.
3. **Открыть мой проект** — статус, следующий шаг, комментарии.

Дополнительные действия — ниже или в меню: галерея, услуги, канал, email, телефон.

## Экраны MVP

### 1. Home / App hub

Задача: быстро направить клиента.

Содержит:
- логотип/маскот Step3D;
- короткий оффер: «Оценим и изготовим 3D-деталь по фото, STL или образцу»;
- 3 крупные кнопки: заявка, файлы, мой проект;
- быстрый Telegram/AI-agent.

Не делать:
- больше 3 равнозначных CTA;
- длинный лендинг внутри Telegram;
- мелкий текст и декоративные стрелки.

### 2. Service choice

Карточки услуг:
- 3D-печать;
- STL/CAD по фото;
- 3D-сканирование;
- реверс-инжиниринг;
- малая серия;
- мастер-класс/образование.

Паттерн из референсов: крупная карточка + простая иконка/маскот + короткое пояснение.

### 3. Project intake

Пошаговая заявка:

1. Что нужно сделать?
2. Какие есть материалы?
3. Размер/количество/срок.
4. Контакт.
5. Отправить.

Ввод:
- текст;
- голос;
- фото;
- STL/STEP/чертёж;
- комментарий.

Кнопка должна называться не `Go`, а **Отправить заявку**.

### 4. Confirmation

После отправки:
- «Заявка принята»;
- номер заявки/проекта;
- что будет дальше;
- кнопки: «Дослать файлы», «Написать AI-агенту», «Открыть мой проект».

### 5. Project card

Карточка проекта:
- статус;
- следующий шаг;
- вводные;
- файлы;
- риски;
- решения;
- ответственный;
- история комментариев.

Важно: настоящие статусы и файлы требуют backend/auth.

### 6. More / Details panel

Использовать нижнюю панель или отдельный экран «Подробнее»:
- что входит в услугу;
- что прислать;
- типичные риски;
- примеры из галереи.

## Айдентика

Из референсов взять:
- пиксельный/техно-маскот Step3D как дружелюбный элемент;
- чёрно-белую базу + синий accent;
- крупные округлые карточки;
- простую систему иконок услуг.

Не брать:
- много разных логотипов одновременно;
- несистемные цвета;
- мелкие рукописные подписи;
- декоративные элементы без функции.

## UX правила

- Один главный CTA на экран.
- Кнопки крупные, подписи прямые.
- Каждый экран отвечает на вопрос: «что мне сейчас сделать?»
- Telegram Mini App должен работать одной рукой.
- Сайт/PWA и Mini App должны вести к одному проектному состоянию.
- Пользователь может начать хаотично; система должна превратить вводные в карточку проекта.

## Архитектура реализации

### Текущий слой

- `Step3D/app/` — PWA communication hub.
- `Step3D/account/` — preview кабинета.
- Google Sheet CRM/CMS — лиды, контент, FAQ, backlog.

### Следующий слой

- Backend API: заявки, статусы, файлы, comments.
- Auth: Telegram initData для Mini App + email magic link для PWA.
- Storage: файлы проекта.
- Admin view: обработка заявок и смена статуса.

### Безопасность

- Не доверять данным из Telegram WebApp без backend-проверки `initData`.
- Не хранить приватные файлы в GitHub Pages.
- Не делать «фейковый кабинет» с персональными данными в статике.

## Definition of Done для Mini App MVP

- Открывается внутри Telegram WebApp.
- Первый экран: 3 главных действия.
- Можно отправить заявку с контактом и описанием.
- Можно дослать файлы через Telegram или backend upload.
- Пользователь получает номер заявки.
- Заявка попадает в CRM/Sheet и локальный журнал.
- Есть карточка проекта хотя бы в MVP-режиме.
- Есть проверка безопасности Telegram initData перед персональными данными.
