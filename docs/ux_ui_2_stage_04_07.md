# UX/UI 2.0 — этапы 4–7

## Этап 4 — честный submit/fallback

- После заявки главный следующий шаг формулируется явно: дослать файлы в Telegram.
- В карточке проекта добавлена кнопка `Скопировать заявку`, чтобы клиент не потерял текст при сбое отправки.
- Telegram deep-link теперь подсказывает: приложить фото/STL/STEP и размеры.

## Этап 5 — file intake

Блок файлов в `Заявке Step3D` стал конкретнее:

- фото: 3–5 ракурсов, масштаб/линейка;
- STL/STEP: исходник, версия, единицы;
- размеры: габарит, посадки, количество;
- контекст: где работает, срок, нагрузка.

Файлы по-прежнему не обязательны до получения номера: это честно отражает текущий статический контур.

## Этап 6 — Viewer как необязательный шаг

- Viewer теперь прямо говорит: если файла нет, можно сразу создать заявку по фото.
- CTA уточнён: `Заявка по фото без Viewer`.
- Калькулятор переименован в `Параметры, если есть модель`, чтобы не выглядеть обязательной частью пути.

## Этап 7 — Account как карточка заявки

- Экран Account в пользовательских местах смещён к формулировке `Карточка заявки`.
- Hero теперь отвечает не «личный кабинет», а «что дальше по проекту Step3D».
- Запрос доступа переименован в запрос карточки проекта.
