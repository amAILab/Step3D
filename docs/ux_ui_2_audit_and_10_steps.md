# Step3D UX/UI 2.0 — 5 экспертных аудитов и 10 этапов

## Общий вывод 5 экспертов

Step3D уже собран как сервисный MVP, но следующий прирост качества — не в «ещё красивее», а в **честном и простом mobile-first пути**:

`Главная → Заявка Step3D → номер/черновик → дослать файлы в Telegram → карточка заявки`

Главные риски:

1. **Доверие после заявки** — нельзя создавать ощущение настоящего кабинета/status tracking без backend/auth.
2. **Момент отправки** — пользователь должен понимать: заявка реально отправлена или это черновик/текст для Telegram.
3. **Файлы** — STL/STEP/фото нужно явно досылать в Telegram с номером проекта.
4. **Viewer** — полезен, но не должен выглядеть обязательным шагом.
5. **Mobile UI** — один главный CTA на экран, меньше конкурирующих кнопок, стабильная нижняя навигация.
6. **Accessibility** — mobile menu, story modal, tabs и form errors требуют усиления.
7. **CSS-системность** — меньше каскадных override-слоёв и `!important`, больше общего UX/UI слоя.

## 10 этапов UX/UI 2.0

### 1. Честный naming и CTA
- App → «Заявка Step3D» / «Оценка проекта» в пользовательских местах.
- Account → «Карточка заявки» / preview, не «личный кабинет».
- Главный CTA: «Рассчитать по фото / STL».

### 2. Mobile-first первый экран
- На главной один основной путь.
- Второй CTA — Telegram/примеры, не равный основному.
- Убрать конкуренцию Viewer/Account на первом экране.

### 3. Заявка как мастер, а не форма
- Контакт + задача — обязательно.
- Файлы — «можно дослать после номера».
- Прогресс понятнее: шаги вместо абстрактного процента.

### 4. Честный submit/fallback
- Success: server/form confirmed `S3D-...`.
- Fail: `S3D-DRAFT-...` и явная кнопка Telegram/email.
- Под кнопкой: «если не отправилось — скопируйте текст и пришлите в Telegram».

### 5. File intake
- Чеклист: 3–5 фото, размер/линейка, количество, срок, назначение, STL/STEP/3MF/OBJ если есть.
- После номера — один большой шаг: «Дослать файлы в Telegram с номером».

### 6. Viewer как необязательный шаг
- Текст: «Не обязательно. Если файла нет — создайте заявку по фото».
- Основной CTA Viewer: «Сохранить расчёт и создать заявку».
- Калькулятор/метрики не должны конкурировать с моделью.

### 7. Карточка заявки / Account preview
- Явный preview-banner: GitHub Pages не хранит приватные файлы и не заменяет менеджера.
- Статус: следующий шаг, что дослать, Telegram инженера.

### 8. Accessibility pass
- Закрытое mobile menu скрыто от скринридера.
- Story modal focus trap.
- Tabs: стрелки/Home/End.
- Form errors: aria-invalid/aria-describedby.
- prefers-reduced-motion.

### 9. Visual system cleanup
- Единый минимальный слой UX/UI tokens.
- Primary CTA чёрный, синий — статус/ссылка/выбор.
- Avatar — главный identity App.

### 10. Release 2.0 audit
- Проверка 320/360/390/430 px.
- Flow smoke: home → app → submit success/fail → viewer → thanks → account.
- Deploy и отчёт.
