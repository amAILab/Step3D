# Session 06 — Account / статус проекта

Цель: сделать `/account/` честным экраном статуса, а не демо-кабинетом. Пользователь должен видеть: номер проекта, что принято, чего не хватает и следующий шаг.

## Что изменено

- Hero стал проще: один CTA `Создать проект`.
- Убраны конкурирующие действия в первом экране:
  - открыть демо-проект;
  - посмотреть структуру;
  - тяжёлый device-preview.
- По умолчанию Account теперь показывает не демо-реестр, а честный preview:
  - `Проект пока не выбран`;
  - `Создайте заявку в App`;
  - статус: ожидаем заявку.
- Если есть `projectId` в URL или `localStorage`, появляется живая карточка:
  - номер `S3D-...`;
  - `Принято`;
  - `Не хватает`;
  - `Дальше`;
  - кнопка `Дослать файлы`;
  - кнопка `Дополнить заявку`.
- Блок файлов переименован в `Что дослать к номеру S3D`.
- Мобильная нижняя панель стала: `Файлы / Заявка`, без демо.
- Account честно остаётся frontend-preview до backend/auth.

## Проверки

- `python3 scripts/project_healthcheck.py` → OK
- `python3 scripts/validate_site.py` → OK
- `python3 scripts/check_analytics_events.py` → OK
- `python3 scripts/check_request_flow.py` → OK
- `python3 scripts/check_backend_templates.py` → OK

## Итог

`/account/` больше не пытается выглядеть как полноценная закрытая CRM. Он стал спокойным статусным экраном: проект есть — показываем следующий шаг; проекта нет — ведём в App.

Следующая сессия — backend-ready MVP: довести контракт и проверки до состояния, где можно подключить реальный endpoint и server-side projectId.
