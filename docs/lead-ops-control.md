# Step3D — контроль заявок и ответов

Цель: чтобы заявки, ответы и проверка работали не «по памяти», а одним повторяемым контуром.

## Один главный чек

Перед публикацией изменений и периодически в работе запускать:

```bash
python3 scripts/check_lead_ops.py
```

Скрипт безопасный: он не отправляет клиентам сообщения, не пишет реальные строки в CRM и работает через self-test / dry-run.

Что проверяет:

1. **Страница заявки** `app/index.html`:
   - контакт обязателен;
   - описание задачи обязательно;
   - номер `S3D-*` создаётся;
   - есть честное предупреждение: файлы не хранятся на GitHub Pages;
   - Telegram используется для досылки тяжёлых файлов;
   - карточка заявки получает `projectId`.
2. **Ответы клиентам**:
   - есть быстрые шаблоны для фото, вопроса о цене и срочной замены;
   - есть правило не обещать цену/срок без проверки;
   - внешние ответы клиентам не отправляются без подтверждения Никиты, кроме заранее разрешённого автоответа.
3. **Маршрутизация заявки**:
   - `lead_router.py --sample --json` выдаёт `S3D-*`, статус, приоритет, SLA и следующий шаг;
   - транспорт остаётся `dry-run`.
4. **CRM**:
   - `append_lead_to_sheet.py --sample --dry-run` показывает строку для таблицы без реальной записи.
5. **Мониторинг почты**:
   - `check_step3d_leads.py --self-test` проверяет черновик ответа без Gmail и без отправки.
6. **Релизные проверки**:
   - `check_request_flow.py`;
   - `check_release_readiness.py`.

## Быстрая ручная проверка заявки

1. Открыть `https://amailab.github.io/Step3D/app/`.
2. Выбрать сценарий: ремонт / прототип / реверс / серия.
3. Заполнить контакт и задачу.
4. Проверить, что справа появился прогресс и следующий шаг.
5. Нажать «Скопировать ТЗ» — текст должен содержать:
   - номер S3D;
   - услугу;
   - контакт;
   - задачу;
   - файлы/ссылки;
   - срок;
   - количество.
6. Проверить кнопку Telegram: она должна открывать чат с подготовленным текстом, а не отправлять без человека.

## Рабочий ритм

- **Ежедневно/перед сменой:** `python3 scripts/check_lead_ops.py`.
- **После изменений сайта:** полный набор:

```bash
python3 scripts/check_lead_ops.py
python3 scripts/validate_site.py
python3 scripts/check_analytics_events.py
python3 scripts/check_seo_trust.py
git diff --check
```

- **Если пришла новая заявка:**
  1. присвоить/сохранить ID `S3D-*`;
  2. определить тип: печать, моделирование по фото, реверс, серия, ивент;
  3. проверить, хватает ли вводных: контакт, фото/файл, размер, количество, срок, назначение;
  4. если данных мало — отправить короткий уточняющий ответ;
  5. если данных достаточно — подготовить маршрут: CAD / печать / сканирование / серия, риски, срок-вилка, цена-вилка.

## Безопасность

- Не обещать цену и срок без проверки файла/фото/размера/нагрузки.
- Не отправлять клиенту финальные обещания без подтверждения Никиты.
- Не хранить токены в репозитории.
- Не считать GitHub Pages хранилищем файлов: тяжёлые файлы — Telegram/облако/почта с номером S3D.
